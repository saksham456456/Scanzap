## 2026-07-10 - Sentinel Security Fixes\n**Vulnerability:** XSS via innerHTML and Replay Attack on Payment API.\n**Learning:** The single-file index.html heavily relied on innerHTML for rendering dynamic lists and history grids which allows XSS. The backend payment verification lacked an idempotency check, allowing duplicate credit issuance via replay attacks.\n**Prevention:** Use document.createElement and textContent/createTextNode instead of innerHTML for DOM updates. In Firebase functions handling payments, always query for existing transaction IDs before processing to ensure idempotency.
